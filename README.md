# 🎵 Criação de Playlist e Privacidade — Casos de Uso

Este repositório contém a documentação de uma atividade desenvolvida na disciplina de **Análise e Projeto de Sistemas**, com foco na modelagem de **Casos de Uso (UML)** para um sistema de gerenciamento de playlists musicais.

O objetivo da atividade é representar as principais funcionalidades relacionadas à criação de playlists e ao controle de privacidade, documentando seus fluxos principais, cenários alternativos, pré e pós-condições.

---

## 🎯 Objetivo da Atividade

* Modelar funcionalidades utilizando **Diagrama de Casos de Uso (UML)**
* Identificar atores e suas interações com o sistema
* Documentar casos de uso seguindo um padrão de especificação
* Definir cenários principais e alternativos
* Aplicar conceitos de Engenharia de Software na modelagem de requisitos funcionais

---

## 🧠 Ferramentas Utilizadas

* **UML (Unified Modeling Language)** – modelagem dos casos de uso
* **Draw.io / Diagrams.net** – elaboração do diagrama
* **Microsoft Word** – documentação dos casos de uso

---

## 📝 Casos de Uso Modelados

| Código | Caso de Uso                 | Descrição                                                                          |
| ------ | --------------------------- | ---------------------------------------------------------------------------------- |
| UC01   | **Fazer Login**             | Permite ao usuário autenticar sua conta para acessar as funcionalidades do sistema |
| UC02   | **Criar Playlist**          | Permite criar uma nova playlist personalizada                                      |
| UC03   | **Adicionar Músicas**       | Permite adicionar músicas à playlist criada                                        |
| UC04   | **Configurar Visibilidade** | Permite definir ou alterar a visibilidade da playlist (Pública ou Privada)         |
| UC05   | **Buscar Músicas**          | Permite localizar músicas por nome, artista ou álbum                               |

## ⚙️ Funcionamento Modelado

O sistema foi modelado considerando o seguinte fluxo principal:

1. O usuário realiza o login no sistema.
2. Cria uma nova playlist.
3. Busca músicas disponíveis.
4. Adiciona as músicas desejadas.
5. Define a visibilidade da playlist (pública ou privada).
6. O sistema salva as informações e confirma a operação.

Também foram documentados cenários alternativos, como:

* Usuário não autenticado;
* Música não encontrada;
* Playlist não encontrada;
* Erro ao salvar alterações de visibilidade.
